# Slide 02 — What Claude Code is

**Layout:** Content
**Background:** paper
**Target duration:** 60 seconds

## Eyebrow (optional, ≤24 chars, all-caps)

POSITIONING

## Headline (sentence case, ≤80 chars)

An agentic coding tool that reads, edits, and ships from your terminal

## Body (3–5 bullets max, OR a key code/command block, OR a comparison table)

- Lives in the terminal, VS Code, JetBrains, Desktop, and the browser — one engine across all surfaces
- Reads your codebase, edits files across many directories, and runs the commands to verify the change
- Connects to external tools through the Model Context Protocol — Drive, Jira, browsers, your own services
- Writes commits, opens pull requests, runs in CI, and can be piped like any Unix tool
- Same `CLAUDE.md` and settings travel with the project across every surface

## Optional: one short quote from the source (≤30 words)

> "Claude Code is composable and follows the Unix philosophy. Pipe logs into it, run it in CI, or chain it with other tools." — Claude Code docs

## Speaker notes (2–3 sentences)

Land the positioning quickly: Claude Code is not an autocomplete plugin, it is an agent that can drive a real engineering task end to end. Stress the surface story — terminal, IDE, web, desktop — because most teams under-use the surfaces they already have. The rest of the deck zooms into the parts that change how people actually work.
